# 北京大事件信息提取工具使用说明

## 常用运行方式

### 1. 同时处理 input 文件 + 增量更新演唱会

```bash
python main.py --concert
```

适用场景：日常更新。

行为：
- 自动读取 `input/` 文件夹里的文件；
- 从文旅部网站增量采集北京演唱会信息；
- 已采集过的演唱会 URL 会跳过；
- 新结果会和当天已有 output Excel 合并、去重、排序，不会直接覆盖旧内容。

---

### 2. 同时处理 input 文件 + 完整重抓演唱会

```bash
python main.py --concert --concert-force
```

适用场景：怀疑演唱会缓存不完整，想重新抓一遍网站数据。

行为：
- 自动读取 `input/` 文件夹里的文件；
- 演唱会部分忽略之前缓存，重新完整抓取；
- 输出仍会和当天已有 output Excel 合并、去重、排序。

注意：`--concert-force` 只影响演唱会采集，不会阻止 input 文件被重新处理。

---

### 3. 只更新演唱会，不处理 input 文件

```bash
python main.py --only-concert
```

适用场景：只想更新文旅部网站演唱会数据。

行为：
- 跳过 `input/` 文件夹；
- 默认按增量模式采集演唱会；
- 输出会和当天已有 output Excel 合并。

如果要只重抓演唱会：

```bash
python main.py --only-concert --concert-force
```

---

### 4. 只处理 input 文件，不更新演唱会

```bash
python main.py --only-input
```

适用场景：只想重新跑本地文件，不访问文旅部网站。

行为：
- 自动读取 `input/` 文件夹里的文件；
- 不采集演唱会；
- 输出会和当天已有 output Excel 合并。

---

## 可选参数

### 指定输入文件夹

```bash
python main.py --concert --input-dir ./input_new
```

### 指定输出文件夹

```bash
python main.py --concert --output-dir ./output_test
```

### 指定演唱会采集日期范围

```bash
python main.py --only-concert --concert-start 2026-01-01 --concert-end 2026-12-31
```

默认范围：过去 2 个月到未来 2 年。

### 指定演唱会场馆规模

```bash
python main.py --only-concert --concert-venue-size large
```

可选值：
- `small`：保留全部，包括小型 LiveHouse；
- `medium`：默认值，排除小型 LiveHouse；
- `large`：只保留大型场馆。

---

## 输出规则

输出文件默认保存在：

```text
output/Travel_Facilitators_and_Hindrances_Events_YYYY.MM.DD.xlsx
```

如果当天文件已经存在，程序会先读取旧内容，再把本次新增内容合并进去，最后统一去重、排序、重新生成 Excel。

---

## 缓存说明

演唱会采集缓存保存在：

```text
state/concert_scrape_state.json
```

普通增量更新会参考这个文件，跳过已经采集过的 URL。

如果使用：

```bash
python main.py --only-concert --concert-force
```

则本次运行会忽略缓存，重新抓取符合条件的演唱会。

---

## 在另一台电脑运行

把整个项目文件夹复制到另一台电脑后，有两种运行方式：

### 方案 A：便携版（推荐，不需要安装 Python）

如果复制的项目文件夹里已经包含 `python/` 和 `run.bat`（自带 Python 运行环境）：

**双击运行：**

```bash
run.bat
```

或带参数运行：

```bash
run.bat --concert
run.bat --only-concert
run.bat --only-input
```

所有参数和上面 `python main.py` 的用法完全一致。

**配置 API Key：**

复制 `.env.example` 为 `.env`，填入你的 DeepSeek API Key：

```env
OPENAI_API_KEY=你的DeepSeek API Key
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-v4-flash
OPENAI_BYPASS_PROXY=1
```

**注意事项：**
- 第一次运行可能需要几分钟初始化浏览器环境；
- 如果杀毒软件提示，选择允许（这是正常的 Python 和浏览器组件）；
- 整个文件夹可以复制到 U 盘或任意位置使用。

---

### 方案 B：完整版（需要安装 Python）

如果复制的是原始项目文件夹（没有自带的 Python）：

#### 1. 安装 Python

建议安装 Python 3.11 或 3.12，至少需要 Python 3.10+。

安装后检查：

```bash
python --version
```

如果 Windows 上 `python` 不可用，可以试：

```bash
py --version
```

#### 2. 安装依赖包

进入项目文件夹后运行：

```bash
python -m pip install -r requirements.txt
```

如果使用的是 Windows 的 `py` 命令：

```bash
py -m pip install -r requirements.txt
```

#### 3. 安装 Playwright 浏览器

演唱会采集需要 Playwright 自动访问文旅部网页。新电脑第一次运行前需要安装 Chromium：

```bash
python -m playwright install chromium
```

如果使用 `py` 命令：

```bash
py -m playwright install chromium
```

#### 4. 配置 `.env`

项目需要 DeepSeek API。确认项目根目录有 `.env` 文件，并包含：

```env
OPENAI_API_KEY=你的DeepSeek API Key
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-v4-flash
OPENAI_BYPASS_PROXY=1
```

注意：`.env` 里有 API Key，不要公开发送给别人。

#### 5. 运行检查

准备完成后，可以先运行：

```bash
python main.py --help
```

如果能正常显示参数说明，再按上面的常用运行方式执行，例如：

```bash
python main.py --concert
```
